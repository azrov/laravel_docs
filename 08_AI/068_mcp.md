# Laravel 12 Dokümantasyonu: Laravel MCP

## Giriş

Laravel MCP, yapay zeka istemcilerinin (AI clients) Model Context Protocol aracılığıyla Laravel uygulamanızla etkileşim kurması için basit ve zarif bir yol sağlar. Uygulamanızla yapay zeka destekli etkileşimleri mümkün kılan sunucular (servers), araçlar (tools), kaynaklar (resources) ve istemler (prompts) tanımlamak için anlamlı, akıcı bir arayüz (expressive, fluent interface) sunar.

## Kurulum (Installation)

Başlamak için, Laravel MCP'yi Composer paket yöneticisini kullanarak projenize kurun:
```bash
composer require laravel/mcp
```

### Rotaları Yayınlama (Publishing Routes)
Laravel MCP'yi yükledikten sonra, MCP sunucularınızı tanımlayacağınız `routes/ai.php` dosyasını yayınlamak için `vendor:publish` Artisan komutunu çalıştırın:
```bash
php artisan vendor:publish --tag=ai-routes
```
Bu komut, uygulamanızın `routes` dizininde MCP sunucularınızı kaydetmek için kullanacağınız `routes/ai.php` dosyasını oluşturur.

## Sunucu (Server) Oluşturma

`make:mcp-server` Artisan komutunu kullanarak bir MCP sunucusu oluşturabilirsiniz. Sunucular, araçlar, kaynaklar ve istemler gibi MCP yeteneklerini yapay zeka istemcilerine sunan merkezi iletişim noktası olarak görev yapar:
```bash
php artisan make:mcp-server WeatherServer
```
Bu komut, `app/Mcp/Servers` dizininde yeni bir sunucu sınıfı oluşturacaktır. Oluşturulan sunucu sınıfı, Laravel MCP'nin temel `Laravel\Mcp\Server` sınıfını genişletir (extend) ve sunucuyu yapılandırmak, araçları, kaynakları ve istemleri kaydetmek için nitelikler (attributes) ve özellikler (properties) sağlar:
```php
<?php

namespace App\Mcp\Servers;

use Laravel\Mcp\Server\Attributes\Instructions;
use Laravel\Mcp\Server\Attributes\Name;
use Laravel\Mcp\Server\Attributes\Version;
use Laravel\Mcp\Server;

#[Name('Hava Durumu Sunucusu')]
#[Version('1.0.0')]
#[Instructions('Bu sunucu hava durumu bilgisi ve tahminleri sağlar.')]
class WeatherServer extends Server
{
    /**
     * Bu MCP sunucusuna kayıtlı araçlar.
     *
     * @var array<int, class-string<\Laravel\Mcp\Server\Tool>>
     */
    protected array $tools = [
        // GetCurrentWeatherTool::class,
    ];

    /**
     * Bu MCP sunucusuna kayıtlı kaynaklar.
     *
     * @var array<int, class-string<\Laravel\Mcp\Server\Resource>>
     */
    protected array $resources = [
        // WeatherGuidelinesResource::class,
    ];

    /**
     * Bu MCP sunucusuna kayıtlı istemler.
     *
     * @var array<int, class-string<\Laravel\Mcp\Server\Prompt>>
     */
    protected array $prompts = [
        // DescribeWeatherPrompt::class,
    ];
}
```

### Sunucu Kaydı (Server Registration)
Bir sunucu oluşturduktan sonra, onu erişilebilir kılmak için `routes/ai.php` dosyanıza kaydetmelisiniz. Laravel MCP, sunucuları kaydetmek için iki yöntem sağlar: HTTP üzerinden erişilebilen sunucular için `web` ve komut satırı sunucuları için `local`.

### Web Sunucuları (Web Servers)
Web sunucuları en yaygın sunucu türüdür ve HTTP POST istekleriyle erişilebilir, bu da onları uzak yapay zeka istemcileri veya web tabanlı entegrasyonlar için ideal kılar. `web` yöntemini kullanarak bir web sunucusu kaydedin:
```php
use App\Mcp\Servers\WeatherServer;
use Laravel\Mcp\Facades\Mcp;

Mcp::web('/mcp/weather', WeatherServer::class);
```
Normal rotalar gibi, web sunucularınızı korumak için ara yazılımlar (middleware) uygulayabilirsiniz:
```php
Mcp::web('/mcp/weather', WeatherServer::class)
    ->middleware(['throttle:mcp']);
```

### Yerel Sunucular (Local Servers)
Yerel sunucular Artisan komutları olarak çalışır ve Laravel Boost gibi yerel yapay zeka asistanı entegrasyonları oluşturmak için mükemmeldir. `local` yöntemini kullanarak bir yerel sunucu kaydedin:
```php
use App\Mcp\Servers\WeatherServer;
use Laravel\Mcp\Facades\Mcp;

Mcp::local('weather', WeatherServer::class);
```
Kaydedildikten sonra, genellikle `mcp:start` Artisan komutunu kendiniz manuel olarak çalıştırmanız gerekmez. Bunun yerine, MCP istemcinizi (yapay zeka ajanınızı) sunucuyu başlatacak şekilde yapılandırın veya MCP Inspector'ı kullanın.

## Araçlar (Tools)

Araçlar (tools), sunucunuzun yapay zeka istemcilerinin çağırabileceği işlevselliği kullanıma sunmasını sağlar. Dil modellerinin eylemler gerçekleştirmesine, kod çalıştırmasına veya harici sistemlerle etkileşime girmesine izin verir:
```php
<?php

namespace App\Mcp\Tools;

use Illuminate\Contracts\JsonSchema\JsonSchema;
use Laravel\Mcp\Request;
use Laravel\Mcp\Response;
use Laravel\Mcp\Server\Attributes\Description;
use Laravel\Mcp\Server\Tool;

#[Description('Belirtilen konum için güncel hava durumu tahminini getirir.')]
class CurrentWeatherTool extends Tool
{
    /**
     * Araç isteğini işle.
     */
    public function handle(Request $request): Response
    {
        $location = $request->get('location');

        // Hava durumunu al...

        return Response::text('Hava durumu...');
    }

    /**
     * Aracın girdi şemasını (input schema) al.
     *
     * @return array<string, \Illuminate\JsonSchema\Types\Type>
     */
    public function schema(JsonSchema $schema): array
    {
        return [
            'location' => $schema->string()
                ->description('Hava durumunun alınacağı konum.')
                ->required(),
        ];
    }
}
```

### Araç Oluşturma (Creating Tools)
Bir araç oluşturmak için `make:mcp-tool` Artisan komutunu çalıştırın:
```bash
php artisan make:mcp-tool CurrentWeatherTool
```
Bir araç oluşturduktan sonra, onu sunucunuzun `$tools` özelliğine kaydedin:
```php
<?php

namespace App\Mcp\Servers;

use App\Mcp\Tools\CurrentWeatherTool;
use Laravel\Mcp\Server;

class WeatherServer extends Server
{
    /**
     * Bu MCP sunucusuna kayıtlı araçlar.
     *
     * @var array<int, class-string<\Laravel\Mcp\Server\Tool>>
     */
    protected array $tools = [
        CurrentWeatherTool::class,
    ];
}
```

#### Araç Adı (Name), Başlığı (Title) ve Açıklaması (Description)
Varsayılan olarak, aracın adı ve başlığı sınıf adından türetilir. Örneğin, `CurrentWeatherTool` `current-weather` adına ve `Current Weather Tool` başlığına sahip olacaktır. Bu değerleri `Name` ve `Title` niteliklerini kullanarak özelleştirebilirsiniz:
```php
use Laravel\Mcp\Server\Attributes\Name;
use Laravel\Mcp\Server\Attributes\Title;

#[Name('get-optimistic-weather')]
#[Title('İyimser Hava Durumu Tahmini Al')]
class CurrentWeatherTool extends Tool
{
    // ...
}
```
Araç açıklamaları otomatik olarak oluşturulmaz. `Description` niteliğini kullanarak her zaman anlamlı bir açıklama sağlamalısınız:
```php
use Laravel\Mcp\Server\Attributes\Description;

#[Description('Belirtilen konum için güncel hava durumu tahminini getirir.')]
class CurrentWeatherTool extends Tool
{
    //
}
```
Açıklama, aracın meta verilerinin kritik bir parçasıdır, çünkü yapay zeka modellerinin aracı ne zaman ve nasıl etkili bir şekilde kullanacağını anlamasına yardımcı olur.

### Araç Girdi Şemaları (Tool Input Schemas)
Araçlar, yapay zeka istemcilerinden hangi argümanları kabul ettiklerini belirtmek için girdi şemaları (input schemas) tanımlayabilir. Aracınızın girdi gereksinimlerini tanımlamak için Laravel'in `Illuminate\Contracts\JsonSchema\JsonSchema` oluşturucusunu (builder) kullanın:
```php
<?php

namespace App\Mcp\Tools;

use Illuminate\Contracts\JsonSchema\JsonSchema;
use Laravel\Mcp\Server\Tool;

class CurrentWeatherTool extends Tool
{
    /**
     * Aracın girdi şemasını al.
     *
     * @return array<string, \Illuminate\JsonSchema\Types\Type>
     */
    public function schema(JsonSchema $schema): array
    {
        return [
            'location' => $schema->string()
                ->description('Hava durumunun alınacağı konum.')
                ->required(),

            'units' => $schema->string()
                ->enum(['celsius', 'fahrenheit'])
                ->description('Kullanılacak sıcaklık birimleri.')
                ->default('celsius'),
        ];
    }
}
```

### Araç Çıktı Şemaları (Tool Output Schemas)
Araçlar, yanıtlarının yapısını belirtmek için çıktı şemaları (output schemas) tanımlayabilir. Bu, ayrıştırılabilir araç sonuçlarına ihtiyaç duyan yapay zeka istemcileriyle daha iyi entegrasyon sağlar. Aracınızın çıktı yapısını tanımlamak için `outputSchema` yöntemini kullanın:
```php
<?php

namespace App\Mcp\Tools;

use Illuminate\Contracts\JsonSchema\JsonSchema;
use Laravel\Mcp\Server\Tool;

class CurrentWeatherTool extends Tool
{
    /**
     * Aracın çıktı şemasını al.
     *
     * @return array<string, \Illuminate\JsonSchema\Types\Type>
     */
    public function outputSchema(JsonSchema $schema): array
    {
        return [
            'temperature' => $schema->number()
                ->description('Santigrat cinsinden sıcaklık')
                ->required(),

            'conditions' => $schema->string()
                ->description('Hava durumu koşulları')
                ->required(),

            'humidity' => $schema->integer()
                ->description('Nem yüzdesi')
                ->required(),
        ];
    }
}
```

### Araç Argümanlarını Doğrulama (Validating Tool Arguments)
JSON Şema tanımları, araç argümanları için temel bir yapı sağlar, ancak daha karmaşık doğrulama kuralları da uygulamak isteyebilirsiniz.

Laravel MCP, Laravel'in doğrulama özellikleriyle sorunsuz bir şekilde entegre olur. Gelen araç argümanlarını, aracınızın `handle` metodu içinde doğrulayabilirsiniz:
```php
<?php

namespace App\Mcp\Tools;

use Laravel\Mcp\Request;
use Laravel\Mcp\Response;
use Laravel\Mcp\Server\Tool;

class CurrentWeatherTool extends Tool
{
    /**
     * Araç isteğini işle.
     */
    public function handle(Request $request): Response
    {
        $validated = $request->validate([
            'location' => 'required|string|max:100',
            'units' => 'in:celsius,fahrenheit',
        ]);

        // Doğrulanmış argümanları kullanarak hava durumu verilerini al...
    }
}
```
Doğrulama başarısız olduğunda, yapay zeka istemcileri sağladığınız hata mesajlarına göre hareket edecektir. Bu nedenle, net ve uygulanabilir (actionable) hata mesajları sağlamak kritik öneme sahiptir:
```php
$validated = $request->validate([
    'location' => ['required','string','max:100'],
    'units' => 'in:celsius,fahrenheit',
],[
    'location.required' => 'Hava durumunu almak için bir konum belirtmelisiniz. Örneğin, "New York City" veya "Tokyo".',
    'units.in' => 'Birimler için "celsius" veya "fahrenheit" belirtmelisiniz.',
]);
```

#### Araç Bağımlılık Enjeksiyonu (Tool Dependency Injection)
Tüm araçları çözümlemek için (resolve) Laravel servis kabı (service container) kullanılır. Sonuç olarak, aracınızın ihtiyaç duyabileceği herhangi bir bağımlılığı kurucusunda (constructor) tip belirtebilirsiniz (type-hint). Bildirilen bağımlılıklar otomatik olarak çözümlenecek ve araç örneğine enjekte edilecektir:
```php
<?php

namespace App\Mcp\Tools;

use App\Repositories\WeatherRepository;
use Laravel\Mcp\Server\Tool;

class CurrentWeatherTool extends Tool
{
    /**
     * Yeni bir araç örneği oluştur.
     */
    public function __construct(
        protected WeatherRepository $weather,
    ) {}

    // ...
}
```
Kurucu enjeksiyonuna ek olarak, aracınızın `handle()` metodunda da bağımlılıkları tip belirtebilirsiniz. Metot çağrıldığında servis kabı bağımlılıkları otomatik olarak çözümleyecek ve enjekte edecektir:
```php
<?php

namespace App\Mcp\Tools;

use App\Repositories\WeatherRepository;
use Laravel\Mcp\Request;
use Laravel\Mcp\Response;
use Laravel\Mcp\Server\Tool;

class CurrentWeatherTool extends Tool
{
    /**
     * Araç isteğini işle.
     */
    public function handle(Request $request, WeatherRepository $weather): Response
    {
        $location = $request->get('location');

        $forecast = $weather->getForecastFor($location);

        // ...
    }
}
```

### Araç Ek Açıklamaları (Tool Annotations)
Yapay zeka istemcilerine ek meta veriler sağlamak için araçlarınızı ek açıklamalarla (annotations) zenginleştirebilirsiniz. Bu ek açıklamalar, yapay zeka modellerinin aracın davranışını ve yeteneklerini anlamasına yardımcı olur. Ek açıklamalar, nitelikler (attributes) aracılığıyla araçlara eklenir:
```php
<?php

namespace App\Mcp\Tools;

use Laravel\Mcp\Server\Tools\Annotations\IsIdempotent;
use Laravel\Mcp\Server\Tools\Annotations\IsReadOnly;
use Laravel\Mcp\Server\Tool;

#[IsIdempotent]
#[IsReadOnly]
class CurrentWeatherTool extends Tool
{
    //
}
```

Mevcut ek açıklamalar şunlardır:

| Ek Açıklama (Annotation) | Tip | Açıklama |
| :--- | :--- | :--- |
| `#[IsReadOnly]` | boolean | Aracın ortamını değiştirmediğini belirtir. |
| `#[IsDestructive]` | boolean | Aracın yıkıcı güncellemeler gerçekleştirebileceğini belirtir (salt okunur değilken anlamlıdır). |
| `#[IsIdempotent]` | boolean | Aynı argümanlarla tekrarlanan çağrıların ek bir etkisinin olmadığını belirtir (salt okunur değilken). |
| `#[IsOpenWorld]` | boolean | Aracın harici varlıklarla etkileşime girebileceğini belirtir. |

Ek açıklama değerleri, boolean argümanlar kullanılarak açıkça ayarlanabilir:
```php
use Laravel\Mcp\Server\Tools\Annotations\IsReadOnly;
use Laravel\Mcp\Server\Tools\Annotations\IsDestructive;
use Laravel\Mcp\Server\Tools\Annotations\IsOpenWorld;
use Laravel\Mcp\Server\Tools\Annotations\IsIdempotent;
use Laravel\Mcp\Server\Tool;

#[IsReadOnly(true)]
#[IsDestructive(false)]
#[IsOpenWorld(false)]
#[IsIdempotent(true)]
class CurrentWeatherTool extends Tool
{
    //
}
```

### Koşullu Araç Kaydı (Conditional Tool Registration)
Araç sınıfınızda `shouldRegister` yöntemini uygulayarak araçları çalışma zamanında koşullu olarak kaydedebilirsiniz. Bu yöntem, uygulama durumuna, yapılandırmasına veya istek parametrelerine bağlı olarak bir aracın kullanılabilir olup olmaması gerektiğini belirlemenize olanak tanır:
```php
<?php

namespace App\Mcp\Tools;

use Laravel\Mcp\Request;
use Laravel\Mcp\Server\Tool;

class CurrentWeatherTool extends Tool
{
    /**
     * Aracın kaydedilip kaydedilmeyeceğini belirle.
     */
    public function shouldRegister(Request $request): bool
    {
        return $request?->user()?->subscribed() ?? false;
    }
}
```
Bir aracın `shouldRegister` yöntemi `false` döndürdüğünde, kullanılabilir araçlar listesinde görünmeyecek ve yapay zeka istemcileri tarafından çağrılamayacaktır.

### Araç Yanıtları (Tool Responses)
Araçlar bir `Laravel\Mcp\Response` örneği döndürmelidir. `Response` sınıfı, farklı yanıt türleri oluşturmak için çeşitli kullanışlı metotlar sağlar:

Basit metin yanıtları için `text` yöntemini kullanın:
```php
use Laravel\Mcp\Request;
use Laravel\Mcp\Response;

/**
 * Araç isteğini işle.
 */
public function handle(Request $request): Response
{
    // ...

    return Response::text('Hava Durumu Özeti: Güneşli, 22°C');
}
```
Araç yürütülmesi sırasında bir hata oluştuğunu belirtmek için `error` yöntemini kullanın:
```php
return Response::error('Hava durumu verileri alınamıyor. Lütfen tekrar deneyin.');
```

#### Birden Çok İçerikli Yanıtlar (Multiple Content Responses)
Araçlar, birden çok `Response` örneğinden oluşan bir dizi döndürerek birden çok içerik parçası döndürebilir:
```php
use Laravel\Mcp\Request;
use Laravel\Mcp\Response;

/**
 * Araç isteğini işle.
 */
public function handle(Request $request): Response
{
    $weatherData = // ...

    return [
        Response::text("Sıcaklık: {$weatherData['temp']}°C"),
        Response::text("Koşullar: {$weatherData['conditions']}"),
        Response::text("Rüzgar: {$weatherData['wind']} km/s"),
    ];
}
```
